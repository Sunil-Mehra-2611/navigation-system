🎯 GOAL: Build a Google Street View-like 360° navigation system

✅ WHAT'S INCLUDED:
   • 360° panorama viewing with Three.js
   • Arrow-based navigation between nodes
   • Real-time multi-user sync via WebSocket
   • PostgreSQL database with TypeORM
   • Full TypeScript implementation
   • Auto-seeding sample data

📁 PROJECT STRUCTURE:
   backend/     - NestJS API + WebSocket gateway
   frontend/    - React + Three.js viewer
   database/    - PostgreSQL (via Docker)

🚀 START THE SYSTEM (3 STEPS):

   STEP 1: Start Database
   ----------------------
   Open Terminal 1:
   > docker-compose up -d

   (If you don't have Docker, install PostgreSQL manually)

   STEP 2: Start Backend
   ---------------------
   Open Terminal 2:
   > cd backend
   > npm install
   > npm run start:dev

   Wait for: "360° Panorama System Started"

   STEP 3: Start Frontend
   ----------------------
   Open Terminal 3:
   > cd frontend
   > npm install --legacy-peer-deps
   > npm start

   Browser will auto-open to http://localhost:3000

🎮 HOW TO USE:
   • Drag mouse to rotate 360° view
   • Click arrow buttons to navigate
   • Open multiple tabs to test real-time sync
   • Check console for WebSocket events

🔧 CONFIGURATION:
   • Backend: http://localhost:3001
   • Frontend: http://localhost:3000
   • Database: localhost:5432
   • WebSocket: ws://localhost:3001/panorama

📊 SAMPLE DATA:
   System auto-creates 4 connected panoramas:
   1 → 2 → 3 ← 4

🐛 TROUBLESHOOTING:
   Database error?
   > docker ps (check if running)
   > docker-compose restart

   Port in use?
   > npx kill-port 3000
   > npx kill-port 3001

   TypeScript errors?
   > npm install --legacy-peer-deps

   WebSocket not connecting?
   > Ensure backend is running
   > Check browser console
